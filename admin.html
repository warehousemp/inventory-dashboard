Cycle Count Review Queue

Admin-only review area. This queue lists cycle counts marked READY_FOR_REVIEW by the warehouse team. Posting inventory adjustments must happen from this Admin page only.
Important: Posting a cycle count creates inventory adjustments based on the variances in the selected count. Do not post test counts unless you intentionally want to test inventory adjustment posting.

Admin Review Decision

Use Mark for Recount when the count may be incorrect and the warehouse needs to recount it. Use Void Cycle Count for test sessions, duplicate sessions, wrong categories, or sessions that should never be posted. Neither action posts inventory adjustments.

Post Cycle Count Adjustments

Posting is permanent inventory activity. Confirm the selected cycle count, enter the person posting, and type POST exactly before submitting.

Reverse Inventory Transaction

Admin-only correction tool. This does not delete or edit the original transaction. It creates a new reversal transaction and updates stock in the opposite direction of the original transaction.
Use only when an inventory transaction was entered incorrectly. The original transaction remains in the audit trail, and the reversal is recorded separately.

Cycle Count Reporting

Admin reporting area for completed cycle counts and incomplete aging cycle counts. Completed counts show POSTED cycle counts. Aging counts show OPEN cycle counts that have not been completed within the selected threshold.
Aging threshold applies only to Incomplete / Aging Cycle Counts. Aging report currently includes OPEN cycle counts only.
