Projects Command Center

This is the central landing page for the Molinari Projects workflow. Use this page to navigate to project creation, setup, production workflow visibility, production schedule planning, active project tracking, accounting draws, documents, and administration. Project Detail opens only after a specific project is selected from Active Projects or a project-linked report.

Current Phase: Navigation Structure Standardized Projects Home now uses the current Projects sidebar pattern with collapsible icon-only navigation and remembered sidebar state.

Current Operational Flow

1. Approved Source Sales Handoff, Leadership, Projects Department, or another approved source.
2. Add New Client Project Create the project shell, stage template, and draw schedule.
3. Project Setup Assign Project Manager, assign Project Coordinator, and complete setup requirements.
4. Production Workflow / Active Projects Use the Production Workflow board for phase visibility and Active Projects to open a selected project workspace.
5. Schedule / Accounting / Documents / Admin Review planned production dates, draws, documents, setup validation, template status, and audit history.

Version 1 Scope

  • ✓ Add New Client Project can start from Sales Handoff, leadership direction, Projects Department, or another approved source.
  • ✓ Add New Client Project creates the project shell, stages, and draw schedule.
  • ✓ Project Manager and Project Coordinator assignment happen during Project Setup after the project record exists.
  • ✓ Project Detail is opened only after selecting a specific project from Active Projects or project-linked reports.
  • ✓ Production Workflow provides phase visibility and controlled phase movement for active projects.
  • ✓ Production Schedule provides planned dates, phase calendars, business-day timelines, and Gantt-style schedule visibility.
  • ✓ Tracks project setup, stage visibility, draw schedules, documents, notes, communications, change orders, and audit history.
  • ✓ Engineering and permitting are handled as workflow status triggers, not separate pages yet.
  • ✓ Sales and leads management are intentionally excluded from Version 1.
